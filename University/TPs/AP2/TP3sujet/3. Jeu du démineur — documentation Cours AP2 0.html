3. Jeu du démineur¶

01 October 2015

Objectifs

Programmation du jeu du démineur.

Moyens mis en oeuvre

  • programmation modulaire
  • récursion
  • traitement d’exceptions

3.1. Présentation du jeu du démineur¶

Est-il nécessaire de présenter ce célèbre jeu ?

Si vous ne le connaissez pas, vous êtes invité

  • à consulter la page Wikipédia consacrée à ce sujet,
  • et à faire quelques parties (mais pas pendant cette séance de TP).

3.2. Préparation du travail¶

3.2.1. Création du projet¶

À faire

À l’aide du matériel fourni dans le premier TP, construisez l’arborescence de fichiers d’un nouveau projet que vous nommerez minesweeper.

3.2.2. Récupération de matériel¶

Un matériel de base vous est fourni.

À faire

Récupérez le fichier minesweeper_material.zip et décompressez le dans le dossier src de votre projet.

Vous devez obtenir

  • un fichier minesweeper.py
  • un fichier graphicalboard.py
  • un dossier icons contenant quelques icones.

3.3. Prise en main du matériel¶

3.3.1. Le module minesweeper¶

Commencez par le module minesweeper.

À faire

Complétez la documentation dans l’en-tête du fichier minesweeper.py.

Puis générez et consultez la documentation de ce module.

À découvrir

Après la directive d’importation from enum import Enum, ce module commence par définir un nouveau type GameState (ou plutôt une nouvelle classe) :

class GameState (Enum):
    """
    A class to define an enumerated type with three values :

    * ``winning``
    * ``losing``
    * ``unfinished``

    for the three state of minesweeper game.
    """
    winning = 1
    losing = 2
    unfinished = 3

Cela permet de définit ce qu’en programmation on appelle un type énuméré. Et dans ce cas, ce type contient trois valeurs : winning, losing et unfinished (les valeurs 1, 2 et 3 qui leur sont associées sont totalement arbitraires). Ce type pourra donc être utilisé pour représenter l’état d’un jeu.

Voici une petite session illustrant l’usage de ce type

>>> import minesweeper
>>> minesweeper.GameState
minesweeper.GameState
>>> minesweeper.GameState.winning
<GameState.winning: 1>
>>> type (minesweeper.GameState.winning)
minesweeper.GameState
>>> minesweeper.GameState.winning == 1
False

Notez que malgré la valeur 1 attribuée à winning dans la définition de GameState, GameState.winning n’est pas le nombre 1.

Question

Comment sont représentés

  • le jeu
  • la grille du jeu
  • les cases

Question

Quels sont les constructeurs publics ? et les sélecteurs ? et les modificateurs ?

Il manque manifestement un sélecteur pour les jeux. Lequel ? Réalisez le !

Avertissement

Vous ne pouvez pour l’instant pas utiliser la plupart des fonctions de ce module puisque la fonction neighborhood n’est pas définie.

3.4. Réalisation du travail¶

3.4.1. Achèvement du module minesweeper¶

À faire

Réalisez la fonction neighborhood. Et testez la sans oublier les cas particuliers des cases dans les coins ou sur les bords.

À faire

Réalisez la fonction get_state.

Auparavant précisez les conditions nécessaires et suffisantes pour qu’un jeu soit

  • gagnant,
  • perdant,
  • inachevé.

À faire

Réalisez la fonction reveal_all_cells_from.

Cette fonction peut aisément être programmée en utilisant un algorithme récursif, dont les appels récursifs s’effectuent sur les cases voisines de la case courante.

3.4.2. Programme principal¶

À faire

En utilisant les deux modules minesweeper et graphicalboard, réalisez dans un fichier graphical_main.py un programme principal qui permet de jouer au démineur via l’interface graphique.

Ce programme devra pouvoir prendre des arguments sur la ligne de commande, donnant la largeur, la hauteur, et le nombre de bombes. Des valeurs par défaut peuvent être attribuées en l’absence de ces arguments.

À faire

En utilisant le seul module minesweeper, réalisez dans un fichier console_main.py un programme principal qui permet de jouer au démineur en mode texte dans un terminal.

Ce programme devra pouvoir prendre des arguments sur la ligne de commande, donnant la largeur, la hauteur, et le nombre de bombes. Des valeurs par défaut peuvent être attribuées en l’absence de ces arguments.
