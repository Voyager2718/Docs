2.1. Tracer une fonction récursive¶

La fonction récursive qui suit, nommée fact_talkative (autrement dit fact bavard), renvoie la factorielle de l’entier qu’on lui passe en paramètre selon l’algorithme récursif vu en cours. Seules les lignes 2, 3, 5 et 9 ont été ajoutées afin qu’en plus de la valeur renvoyée, soient imprimés tous les appels récursifs effectués ainsi que les valeurs renvoyées par ces différents appels. Pour cela un paramètre supplémentaire (depth) a aussi été ajouté avec une valeur par défaut égale à 0.

def fact_talkative (n,depth=0):
    dots = '...' * depth
    print ('{:s}->fact({:d})'.format(dots,n))
    if n == 0:
        print ('{:s}<-{:d}'.format(dots,n))
        return 1
    else:
        res =  n * fact_talkative (n - 1,depth + 1)
        print ('{:s}<-{:d}'.format(dots,res))
        return res

La ligne 2 définit une variable locale nommée dots dont la valeur est une chaîne de caractères constituée de trois points répétés depth fois.

La ligne 3 imprime un appel à la fonction fact. Par exemple, avec n valant 4, et depth valant 2, on a

>>> dots = '...' * depth
>>> dots
'......'
>>> print ('{:s}->fact({:d})'.format (dots,n))
......->fact(4)

Les lignes 5 et 9 permettent d’imprimer les valeurs que la fonction va renvoyer. Par exemple, toujours avec les mêmes valeurs des paramètres n et depth, on a

>>> print ('{:s}<-{:d}'.format (dots,res))
......<-24

Et voici ce que donne un appel à cette fonction avec la valeur 6 pour n et la valeur par défaut 0 pour depth

>>> fact_talkative (4)
->fact(6)
...->fact(5)
......->fact(4)
.........->fact(3)
............->fact(2)
...............->fact(1)
..................->fact(0)
..................<-0
...............<-1
............<-2
.........<-6
......<-24
...<-120
<-720
720

On trace ainsi le calcul de 6! en visualisant l’appel initial ainsi que les six appels récursifs successifs, puis les sept valeurs renvoyées par ces différents appels.

  • la somme de deux nombres,
  • le prédicat is_palindromic
  • la fonction binomial.

2.2. Nombres de Fibonacci¶

La suite des nombres de Fibonacci est définie par ses deux premiers termes f0=0: et f1=1 et pour tout entier n≥2 par la relation de récurrence

fn=fn−1+fn−2.

À réaliser

Programmez une fonction récursive qui pour un entier n∈N passé en paramètre renvoie le terme fn de la suite de Fibonacci.

Vérifiez la validité de votre programme en calculant les nombres fn pour n compris entre 0 et 10. (pour cela vous devrez préalablement calculer «à la main» ces nombres.)

À réaliser

Utilisez votre programme pour calculer f40. Que constatez-vous ?

À réaliser

Afin d’avoir une idée de la quantité de calculs nécessaires pour calculer un nombre de Fibonacci avec votre programme, réalisez une version bavarde de votre fonction.

Vous allez utiliser la version bavarde de votre fonction pour déterminer expérimentalement le nombre d’appels à la fonction nécessaires aux calculs des nombres de Fibonacci.

À réaliser

Pour cela, ajoutez ces quelques lignes de code (qui supposent que votre fonction bavarde se nomme fibo_talkative) à la fin de votre fichier :

if __name__ == "__main__":
    import sys
    n = int (sys.argv[1])
    print (fibo_talkative (n))

puis, en supposant que votre fichier se nomme fibonacci.py, dans un terminal de commandes, tapez :

$ python3 fibonacci.py 5

vous devez obtenir

->fibo(4)
...->fibo(3)
......->fibo(2)
.........->fibo(1)
.........<-1
.........->fibo(0)
.........<-0
......<-1
......->fibo(1)
......<-1
...<-2
...->fibo(2)
......->fibo(1)
......<-1
......->fibo(0)
......<-0
...<-1
<-3
3

On constate que la commande a pour résultat l’affichage de 19 lignes, dont la dernière est l’impression du nombre de Fibonacci f4=3, et les 18 précédentes sont l’impression des neuf appels à la fonction et les neuf valeurs que ces appels renvoient.

Question

Quel rapport existe-t-il entre le nombre de lignes imprimées et le nombre d’appels à la fonction ?

On peut déterminer le nombre de lignes imprimées en redirigeant la sortie produite par la commande vers l’entrée de la commande wc -l qui détermine le nombre de lignes qu’on lui donne :

$ python3 fibonacci.py 4 | wc -l
19

À réaliser

Utilisez cette commande pour déterminer le nombre d’appels à la fonction dans le calcul de fn pour n compris entre 0 et 10, puis pour n=40.
